# Permissions

Permissions define what apps can do through SPMT.

## Example Scopes

- `identity:read`
- `apps:read`
- `apps:write`
- `messages:read`
- `messages:write`
- `events:write`
- `athena:write`
- `webhooks:write`

## Principles

- Ask for the minimum permissions.
- Explain permissions in user-friendly language.
- Keep privileged server actions behind scoped keys.
- Let users see what an app can access.
